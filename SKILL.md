# 三省六部任务调度器 - 通用版

## 🏛️ 技能概述

这是一个基于古代三省六部制的通用任务调度系统。当用户明确说「使用三省六部」时，系统会启动这个调度器，在后台异步执行完整的三省六部流程，同时不阻塞聊天窗口。

---

## 🎯 核心设计原则

1. **异步执行** - 使用 sessions_spawn 创建子任务，不阻塞主会话
2. **立即回复** - 接到任务后立即回复，然后 sessions_yield 释放聊天窗口
3. **角色人格** - 每个子任务都有明确的角色设定（从配置读取）
4. **并行执行** - 六部任务同时执行，最大化效率
5. **配置驱动** - 所有参数从配置文件读取，方便自定义

---

## 📋 执行流程

```
用户指令（包含触发词）
    │
    ▼
系统立即回复「收到！正在启动三省六部处理...」
    │
    ▼
sessions_yield() 释放聊天窗口
    │
    ▼
创建异步子任务（后台执行）
    │
    ├── Step 1: 中书省规划（方案设计）
    │   └─ 角色：老成持重，擅长规划
    │
    ├── Step 2: 门下省审核（审核把关）
    │   └─ 角色：严谨挑剔，善于发现问题
    │
    ├── Step 3: 尚书省派发（任务协调）
    │   └─ 角色：执行力强，务实干练
    │
    └── Step 4: 六部并行执行
        ├─ 兵部：搜索、基础设施
        ├─ 户部：价格、成本、数据
        ├─ 礼部：文档、规范
        ├─ 工部：技术实现
        └─ 刑部：合规、审计
            │
            ▼
        汇总结果
            │
            ▼
系统接收结果 → 发送新消息呈现
```

## 🏛️ 角色设定（从配置读取）

### 三省

| 部门 | 性格 | 职责 |
|------|------|------|
| **中书省** | 老成持重，擅长规划 | 方案规划与流程设计 |
| **门下省** | 严谨挑剔，善于发现问题 | 方案审核与把关 |
| **尚书省** | 执行力强，关注可行性 | 任务派发与执行协调 |

### 六部

| 部门 | 性格 | 职责 |
|------|------|------|
| **兵部** | 雷厉风行 | 搜索、基础设施、应急响应 |
| **户部** | 精打细算 | 价格、成本、数据分析 |
| **礼部** | 文采飞扬 | 文档、规范、对外沟通 |
| **工部** | 技术宅 | 工程实现、架构设计 |
| **刑部** | 严明公正 | 合规、审计、质量把控 |

## ⚙️ 配置文件说明

所有配置从 `config/settings.json` 读取：

### 可配置项

| 配置项 | 说明 |
|--------|------|
| `trigger_words` | 触发词（中英文） |
| `roles.*.title` | 部门名称 |
| `roles.*.alias` | 部门别名 |
| `roles.*.identity` | 身份描述 |
| `roles.*.duty` | 职责描述 |
| `roles.*.timeout` | ⚠️ 参考超时时间（秒），仅作文档参考 |
| `roles.*.enabled` | 是否启用 |
| `roles.*.order` | 执行顺序 |

### ⚠️ timeout 说明

**timeout 是参考值，不是强制执行的！**

- 这个值用于文档和给开发者一个预期时间
- 实际执行时的超时由 OpenClaw 的 `sessions_spawn` 的 `timeoutSeconds` 参数控制
- 你可以随时在代码中覆盖这个值
| `roles.*.order` | 执行顺序 |
| `execution.parallel_execution` | 是否并行 |

### 自定义示例

```json
{
  "trigger_words": {
    "zh": ["使用我的委员会"]
  },
  "roles": {
    "my_department": {
      "title": "我的部门",
      "alias": "部长",
      "identity": "部门负责人",
      "personality": "专业",
      "style": "分析中",
      "duty": "自定义职责",
      "timeout": 120,
      "enabled": true,
      "order": 1
    }
  }
}
```
