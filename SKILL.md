# 三省六部通用任务处理器

## 🏛️ 技能概述

这是一个基于古代三省六部制的异步任务调度系统。当用户明确说「使用三省六部」等触发词时，Agent 会启动这个技能，agent在后台异步执行完整的三省六部流程，同时不阻塞聊天窗口。

---

## 🎯 核心设计原则

1. **异步执行** - 使用 sessions_spawn 创建子任务，不阻塞主会话
2. **立即回复** - 接到触发词后立即回复，然后 sessions_yield 释放聊天窗口
3. **角色人格** - 每个子任务都有明确的角色设定和说话风格，可选指定模型
4. **并行执行** - 六部任务同时执行，最大化效率
5. **关键节点汇报** - 只在关键阶段汇报进度，避免打扰

---

## 📋 执行步骤（详细操作指南）

**当接收到用户消息时，按以下步骤执行：**

### 步骤 0：检测触发词

**操作方法：**
1. 读取用户消息内容
2. 检查消息中是否包含以下任一触发词：
   - 「使用三省六部」
   - 「启动三省六部」
   - 「调用三省六部」
   - 「三省六部帮我」
   - 「用三省六部」

**判断结果：**
- 如果包含触发词 → **继续执行步骤 1**
- 如果不包含触发词 → **不执行本技能，按正常流程处理用户消息**

---

### 步骤 1：立即回复

**目的：** 告知用户任务已接收，缓解等待焦虑

**操作方法：**
1. agent调用 `message` 工具
2. 设置参数：
   ```
   action: "send"
   targets: ["当前用户"]
   message: "收到！正在安排三省六部处理，稍后向您汇报结果～"
   ```
3. 发送消息

**预期结果：** 用户收到确认消息

---

### 步骤 2：释放聊天窗口

**目的：** 不阻塞聊天窗口，让用户可以继续其他操作

**操作方法：**
1. agent调用 `sessions_yield()` 工具
2. 不传递任何参数
3. 结束当前回合

**预期结果：** 聊天窗口释放，用户可以继续发送消息

**重要：** 此步骤必须执行，否则用户无法继续聊天

---

### 步骤 3：创建太子子任务（消息分拣）

**目的：** 将用户任务拆解为明确的需求清单

**操作方法：**
1. agent调用 `sessions_spawn` 工具
2. 设置参数：
   ```
   task: "你是太子，负责消息分拣。任务：{用户的原始消息内容}"
   timeoutSeconds: 60
   ```
3. agent等待子任务完成
4. agent获取子任务的返回结果，保存为变量 `taizi_result`

**太子子任务的输出内容：**
- 用户需求的明确清单
- 任务的关键要点
- 需要关注的重点

**示例输出：**
```
太子分拣结果：
1. 用户需求：调研竞品分析工具
2. 关注点：功能对比、价格、用户体验
3. 输出格式：对比表格 + 推荐建议
```

---

### 步骤 4：创建中书省子任务（方案规划）

**目的：** 制定详细的执行方案

**操作方法：**
1. agent调用 `sessions_spawn` 工具
2. 设置参数：
   ```
   task: "你是中书令，负责方案规划。基于以下太子分拣结果：{taizi_result}，请制定详细的执行方案，包括具体步骤、时间安排、资源需求等"
   timeoutSeconds: 120
   ```
3. agent等待子任务完成
4. agent获取子任务的返回结果，保存为变量 `zhongshu_result`

**中书省子任务的输出内容：**
- 详细的执行步骤
- 时间安排
- 资源需求
- 预期成果

---

### 步骤 5：创建门下省子任务（方案审核）

**目的：** 审核方案的可行性和完整性

**操作方法：**
1. agent调用 `sessions_spawn` 工具
2. 设置参数：
   ```
   task: "你是侍中，负责方案审核。请审核以下中书省规划方案：{zhongshu_result}。审核要点：1) 方案是否可行 2) 是否有遗漏 3) 是否有风险。如果发现问题，请明确指出并提出修改意见；如果没问题，请明确回复'审核通过，可以执行'"
   timeoutSeconds: 90
   ```
3. agent等待子任务完成
4. agent获取子任务的返回结果，保存为变量 `menxia_result`

**审核判断标准：**
1. 检查 `menxia_result` 中是否包含以下关键词：
   - 否定关键词："不通过"、"有问题"、"需要修改"、"建议调整"、"存在风险"、"不可行"
   - 肯定关键词："审核通过"、"可以执行"、"没问题"、"通过"、"可行"

2. 判断逻辑：
   - 如果包含否定关键词 → **执行步骤 5A（重新规划）**
   - 如果包含肯定关键词 → **执行步骤 6**

---

### 步骤 5A：重新规划（审核不通过时执行）

**目的：** 根据门下省的修改意见重新规划方案

**操作方法：**
1. 设置重新规划计数器 `retry_count = 0`
2. 循环执行以下操作（最多 3 次）：
   
   a. agent调用 `sessions_spawn` 工具
   b. 设置参数：
      ```
      task: "你是中书令，请根据门下省的修改意见：{menxia_result}，重新规划方案。原方案：{zhongshu_result}"
      timeoutSeconds: 120
      ```
   c. agent等待子任务完成
   d. agent获取新结果，更新 `zhongshu_result`
   
   e. agent再次调用 `sessions_spawn` 工具进行复审
   f. 设置参数：
      ```
      task: "你是侍中，请再次审核中书省的新方案：{zhongshu_result}。如果通过请回复'审核通过'，如仍有问题请继续提出修改意见"
      timeoutSeconds: 90
      ```
   g. agent获取复审结果，更新 `menxia_result`
   
   h. agent检查复审结果：
      - 如果包含肯定关键词 → 退出循环，执行步骤 6
      - 如果不包含肯定关键词 → `retry_count + 1`，继续循环
   
   i. 如果 `retry_count >= 3` → 退出循环，执行步骤 6（使用当前方案）

---

### 步骤 6：创建尚书省子任务（任务派发）

**目的：** 将方案分解为六部的具体任务

**操作方法：**
1. agent调用 `sessions_spawn` 工具
2. 设置参数：
   ```
   task: "你是尚书令，负责任务派发。请根据审核通过的方案：{menxia_result}，将任务分解为六个部门的具体指令：1) 兵部（搜索、基础设施）2) 户部（价格、成本、数据）3) 礼部（文档、规范）4) 工部（工程实现、架构）5) 刑部（合规、审计、质量）6) 吏部（人事、协调）。请为每个部门明确任务内容和输出要求"
   timeoutSeconds: 60
   ```
3. agent等待子任务完成
4. agent获取子任务的返回结果，保存为变量 `shangshu_result`

**尚书省子任务的输出内容：**
- 六部各自的任务清单
- 每个部门的输出要求
- 任务之间的协调关系

---

### 步骤 7：创建六部子任务（并行执行）

**目的：** 六部分别执行具体任务

**操作方法：**
1. agent为每个部门调用 `sessions_spawn` 工具，创建 6 个子任务（并行执行）

2. **兵部子任务：**
   ```
   task: "你是兵部尚书，负责搜索和基础设施。请根据尚书省派发的指令：{shangshu_result}，执行你的任务，输出搜索结果和基础设施建议"
   timeoutSeconds: 180
   ```

3. **户部子任务：**
   ```
   task: "你是户部尚书，负责价格、成本、数据分析。请根据尚书省派发的指令：{shangshu_result}，执行你的任务，输出价格对比、成本分析和数据统计"
   timeoutSeconds: 180
   ```

4. **礼部子任务：**
   ```
   task: "你是礼部尚书，负责文档和规范。请根据尚书省派发的指令：{shangshu_result}，执行你的任务，输出规范文档和沟通建议"
   timeoutSeconds: 180
   ```

5. **工部子任务：**
   ```
   task: "你是工部尚书，负责工程实现和架构设计。请根据尚书省派发的指令：{shangshu_result}，执行你的任务，输出技术方案和架构设计"
   timeoutSeconds: 180
   ```

6. **刑部子任务：**
   ```
   task: "你是刑部尚书，负责合规、审计、质量把控。请根据尚书省派发的指令：{shangshu_result}，执行你的任务，输出合规检查、审计结果和质量评估"
   timeoutSeconds: 180
   ```

7. **吏部子任务：**
   ```
   task: "你是吏部尚书，负责人事管理和团队协调。请根据尚书省派发的指令：{shangshu_result}，执行你的任务，输出人员安排和协调建议"
   timeoutSeconds: 180
   ```

8. agent等待所有 6 个子任务完成
9. agent获取每个子任务的返回结果，分别保存为：
   - `bingbu_result`
   - `hubu_result`
   - `libu_result`
   - `gongbu_result`
   - `xingbu_result`
   - `libu_hr_result`

---

### 步骤 8：汇总结果

**目的：** 将三省六部的执行结果汇总为完整报告

**操作方法：**
1. agent收集所有结果变量：
   - `taizi_result`（太子分拣结果）
   - `zhongshu_result`（中书省规划结果）
   - `menxia_result`（门下省审核结果）
   - `shangshu_result`（尚书省派发结果）
   - `bingbu_result`（兵部执行结果）
   - `hubu_result`（户部执行结果）
   - `libu_result`（礼部执行结果）
   - `gongbu_result`（工部执行结果）
   - `xingbu_result`（刑部执行结果）
   - `libu_hr_result`（吏部执行结果）

2. agent提取每个结果的摘要（前 200 字或关键段落）

3. 按以下格式组装报告：
   ```
   📋 三省六部任务完成报告

   📝 任务：{用户原始任务}
   ✅ 状态：已完成

   🏛️ 三省执行摘要：
   - 太子：{taizi_result 摘要}
   - 中书省：{zhongshu_result 摘要}
   - 门下省：{menxia_result 摘要}
   - 尚书省：{shangshu_result 摘要}

   📊 六部执行结果：
   - 兵部：{bingbu_result 摘要}
   - 户部：{hubu_result 摘要}
   - 礼部：{libu_result 摘要}
   - 工部：{gongbu_result 摘要}
   - 刑部：{xingbu_result 摘要}
   - 吏部：{libu_hr_result 摘要}

   💡 总结：{整体总结和建议}
   ```

---

### 步骤 9：回奏用户

**目的：** 向用户发送完整的执行结果

**操作方法：**
1. agent调用 `message` 工具
2. 设置参数：
   ```
   action: "send"
   targets: ["当前用户"]
   message: "{步骤 8 组装的完整报告}"
   ```
3. 发送消息

**预期结果：** 用户收到完整的三省六部执行报告

## ⚠️ 注意事项

1. **必须使用触发词** - 只有明确说「使用三省六部」才触发
2. **必须 sessions_yield** - 回复后立即释放聊天窗口
3. **超时设置** - 每个子任务设置合理超时（60-180 秒）
4. **错误处理** - 子任务失败的处理方法：
   - 如果太子、中书、门下、尚书中任意一个失败，终止整个流程，向用户发送错误报告
   - 如果六部中某一部失败，继续执行其他部门，汇总时标注失败部门
   - 错误报告格式：「⚠️ 执行失败：{失败部门} - {失败原因}」
5. **审核循环** - 门下省审核不通过时重新规划，最多 3 次
6. **数据传递** - 确保每个步骤的结果正确传递给下一步

## ⚙️ 配置文件说明

配置文件路径：`config/settings.json`

### 配置结构
```json
{
  "trigger_words": {
    "zh": ["使用三省六部", "启动三省六部", "调用三省六部", "三省六部帮我", "用三省六部"],
    "en": ["use sansheng liubu", "start三省六部", "invoke analysis committee"]
  },
  "roles": {
    "taizi": { "title": "太子", "personality": "年轻有为", "style": "本宫以为", "duty": "消息分拣", "timeout": 60 },
    "zhongshu": { "title": "中书令", "personality": "老成持重", "style": "需从三方面考量", "duty": "方案规划", "timeout": 120 },
    ...  
  },
  "models": {
    "taizi": "qwen/qwen3.5-plus",
    "zhongshu": "qwen/qwen3.6-plus",
    "department_default": "qwen/qwen3.5-plus"
  }
}
```

### 模型配置使用

如果需要在 `sessions_spawn` 中指定模型，添加 `model` 参数：
```
sessions_spawn(
    task: "任务描述",
    timeoutSeconds: 60,
    model: "qwen/qwen3.6-plus"  // 可选参数
)
```

### 配置优先级
1. 如果 settings.json 中配置了模型，优先使用配置的模型
2. 如果未配置，使用用户的默认模型
