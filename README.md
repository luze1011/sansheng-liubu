## 📖 简介

这是一个基于古代三省六部制的**OpenClaw 技能**。当用户明确说触发词时，Agent 会启动三省六部流程，在后台异步执行多角色协作任务，同时不阻塞聊天窗口。

**技能特点：**
- ✅ 异步执行 - 不阻塞聊天
- ✅ 多角色协作 - 三省 + 六部
- ✅ 配置驱动 - 支持自定义角色和模型
- ✅ 安装即用 - 无需修改

---

## 🏛️ 三省六部架构

### 三省（串行执行）

| 部门 | 职责 | 性格 |
|------|------|------|
| **太子** | 消息分拣 | 年轻有为 |
| **中书省** | 方案规划 | 老成持重 |
| **门下省** | 审核把关 | 严谨挑剔 |
| **尚书省** | 任务派发 | 执行力强 |

### 六部（并行执行）

| 部门 | 职责 |
|------|------|
| **兵部** | 搜索、基础设施 |
| **户部** | 价格、成本、数据 |
| **礼部** | 文档、规范 |
| **工部** | 工程实现、架构 |
| **刑部** | 合规、审计 |
| **吏部** | 人事、协调 |

## 🎯 使用方法

### 触发示例

```
使用三省六部帮我分析竞品
启动三省六部来处理这个任务
调用三省六部分析市场趋势
三省六部帮我规划项目
用三省六部评估风险
```

### 系统响应

1. **立即回复**：「收到！正在安排三省六部处理，稍后向父皇汇报结果～」
2. **释放窗口**：调用 sessions_yield() 释放聊天窗口，用户可以继续聊天，不阻塞
3. **异步执行**：后台执行完整流程：
   - 太子分拣 → 中书省规划 → 门下省审核（如不通过返回中书重新规划）→ 尚书省派发 → 六部并行执行
4. **完成回奏**：完成后主动发送详细结果

## ⚙️ 配置说明

所有配置从 `config/settings.json` 读取（可选）：

### 触发词配置

```json
{
  "trigger_words": {
    "zh": ["使用三省六部", "启动三省六部"],
    "en": ["use sansheng liubu"]
  }
}
```

### 角色配置

```json
{
  "roles": {
    "zhongshu": {
      "title": "中书省",
      "alias": "中书令",
      "personality": "老成持重，擅长规划",
      "style": "需从三方面考量",
      "duty": "方案规划",
      "timeout": 120
    }
  }
}
```

### 模型配置（可选）

```json
{
  "models": {
    "taizi": "qwen/qwen3.5-plus",
    "zhongshu": "qwen/qwen3.6-plus",
    "department_default": "qwen/qwen3.5-plus"
  }
}
```

**注意：** 配置文件是可选的。如果不创建配置文件，技能会使用内置默认配置。

## 🔧 自定义配置

如需修改触发词、角色设定或模型配置，创建 `config/settings.json`：

```json
{
  "trigger_words": {

  },
  "roles": {
    "my_department": {

    "zhongshu": "qwen/qwen3.6-plus",
    "department_default": "qwen/qwen3.5-plus"
  }
}
```

1. **触发词必须明确** - 只有说触发词才启动

4. **模型配置可选** - 可指定不同部门使用不同模型
